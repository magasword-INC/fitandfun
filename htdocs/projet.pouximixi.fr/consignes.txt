APPLICATIONS WEB • PROJET SUPERVISÉ
I. OBJECTIF GLOBAL DU PROJET
Faire concevoir et développer une application web simple répondant à une demande client simulée, en
abordant :
▪ Les concepts réseau (serveur, client, IP, DNS, port, requêtes HTTP, etc.)
▪ Les bases de données (modélisation, SQL, requêtes, intégration)
▪ Le développement web (frontend / backend)
▪ La mise en production (serveur web, hébergement local)
▪ La gestion de projet (analyse du besoin, maquette, planification, répartition des tâches)

2. Contexte
Fit&Fun est une association sportive locale proposant différentes activités (fitness, zumba, renforcement
musculaire, yoga…).
L’association compte environ 120 adhérents et 4 animateurs.
La gestion se fait aujourd’hui sur papier et par e-mail, ce qui devient difficile à maintenir : inscriptions,
plannings, cotisations, etc.
Julie souhaite donc faire développer une petite application web qui permettra :
▪ aux adhérents de s’inscrire aux activités et consulter le planning,
▪ aux animateurs d’ajouter ou modifier des séances,
▪ au bureau de l’association de gérer les adhérents.
BTS SIO | Applications Web • Projet supervisé 2
3. Objectifs du projet
Créer une application web simple, accessible depuis ordinateur ou smartphone, permettant :
▪ La gestion des adhérents
o Enregistrement des adhérents (nom, prénom, e-mail, activité choisie, cotisation)
o Consultation / modification / suppression d’un adhérent
▪ La gestion des activités
o Liste des activités proposées (nom, jour, animateur)
o Ajout / suppression / modification d’une activité
▪ Le planning
o Visualisation des créneaux d’activités
o (Optionnel) affichage sous forme de calendrier
▪ L’espace public
o Page d’accueil présentant l’association
o Formulaire d’inscription en ligne (envoi au bureau)
4. Utilisateurs de l’application
Profil Droits / Attentes principales
Adhérent Consulter le planning, s’inscrire à une activité
Animateur Modifier les séances qu’il encadre
Membre du bureau Gérer les adhérents et les activités
Visiteur public Découvrir l’association, envoyer une demande d’inscription
5. Contraintes et attentes techniques
▪ L’application doit être hébergée sur un serveur local de l’association
▪ Elle doit utiliser une base de données relationnelle (MySQL, MariaDB ou PostgreSQL)
▪ Le site doit être accessible via un navigateur web sur le réseau local (ex. http://fitandfun.local)
▪ L’interface doit être claire et intuitive, sans design avancé
